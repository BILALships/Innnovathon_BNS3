# Competition Themes
Teams select one of the two themes below at registration (Milestone J0). This choice is final and guides evaluation throughout the competition.

Embedded AI & Software for Autonomous Robots
This thematic includes:
Environment perception and understanding
Machine Learning and adaptation
Autonomous decision-making
Robotic frameworks (e.g, RTOS)
Embedded Software
Integration between AI, sensors and actuators
Connected Robots & Collective Intelligence
This thematic covers:
Data-exchange (IOT, 5G, wireless communication)
Multi-robot collaboration
Swarm robotics
Distributed and coordinated systems

Note: multiple teams may select the same theme. The jury adapts its evaluation criteria according to the declared theme.

# Recommended Tools and Frameworks

Modular Robotics Development
ROS and ROS 2 enable modular design and integration of perception, planning, and control in robotics systems.
Simulation Environments
Gazebo, Webots, and CoppeliaSim provide realistic virtual robot design and algorithm testing environments.
AI and Perception Tooling
Python with OpenCV, PyTorch, and TensorFlow supports AI pipelines, optimized by TensorRT and OpenVINO accelerators.
Communication Protocols for Robotics and IoT
Wired and wireless communication protocols commonly used in robotic and distributed embedded systems include: Zigbee (Recommended) UART, SPI, I²C, CAN, Ethernet, Bluetooth, Wi‑Fi, 4G/LTE, and 5G.

# Deliverables package – Step by step checklist
MILESTONE J0 – Registration & Declaration
☐  Team information sheet (team name, members, institution) – PDF – J0
☐  Level declaration: A (physical prototype) or B (simulation) – PDF – J0
☐ Declared theme selected from the 6 proposed themes – PDF – J0
MILESTONE J1 – Design Dossier (Phase 1 – all teams)
☐ Functional analysis (objectives, blocks, constraints – MoSCoW) – PDF + PPTX/DOCX – J1
☐ System architecture & block diagram – PDF + source (.drawio / .pptx) – J1
☐ Preliminary BOM (components + estimated costs) – PDF + XLSX – J1
☐ Project Gantt chart & risk matrix – PDF + XLSX – J1☐ Sustainability note (materials, energy, lifecycle) – PDF – J1
☐ Design pitch (≤ 10 slides) + optional 90–120 s concept video – PPTX + MP4 (opt.) – J1
MILESTONE J2–J5 – Semi-Final & Final (Phase 2 – shortlisted teams only)
Source code repository (README, execution instructions, open‑source licenses) – ZIP + GitHub/GitLab link – J2
☐ Demo video (prototype A or simulation B) with metrics & scenarios – MP4 H.264 + thumbnail JPG – J2
☐ Final BOM + Safety File pack – PDF + XLSX – J4
   Safety File pack must include:
   • Hazard analysis + mitigations (mechanical/electrical/battery/software)
   • Power limits + emergency stop strategy + safe start/stop procedure
   • Demo procedure (setup/run/stop/fallback) + operator roles
   • Verification checklist (pre-demo checks, pass/fail criteria)
   Rule: missing/failed Safety File → demo restriction on J5 and Safety score = 0/7
☐ A1 or one‑pager poster (project summary, KPIs, QR code to demo) – high‑resolution PDF – J5
☐ Final pitch (≤ 10 min) presented to the jury + Q&A session – PPTX + PDF – J5
Mandatory file naming:  TeamName_Stage_Tag.extExample: AlphaBot_J1_FunctionalAnalysis.pdf | AlphaBot_J2_DemoVideo.mp4
Security / confidentiality (mandatory)
☐ Default classification: Company Confidential for all files (slides, PDFs, videos, posters, BOM, datasets)
☐ Repositories must be private during the competition; do not publish links publicly before organizer approval
☐ Do not include any third‑party confidential data; document open‑source licenses in README
